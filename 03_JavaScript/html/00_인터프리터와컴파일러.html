컴파일러(compiler) 인터프리터 (interpreter)

        기본 프로그램의 동작을 사용자의 로그에 맞게 수행되도록 해주는 용도의 언어

        별도의 소스 코드 컴파일(이진수로 번역)을 하지 않고 
        Interpreter(인터프리터, 한 줄씩 코드를 읽어서 해석하는 기계, 기능)

        장점 : 컴파일에 소요되는 시간이 없음
        단점 : 프로그램 실행 시간이 상대적으로 느림

        프로그래밍 언어로 작성한 코드를 컴퓨터가 이해할 수 있는 형태로 변환해주는 프로그램
        컴파일러(compiler) : 한국어로 모으다 모임을 구성하다
                            코드 모음 코드 수집
                            사람이 사용하는 언어로 코딩을 진행한 전체 소스 코드를
                            한 번에 컴퓨터가 알아들을 수 있는 언어로 변환하고 변환이 완료된
                            파일을 생성하여 파일을 실행
                            사람이만든파일.java → 사람이만들파일.class
                            실행할 때는 사람이만든파일.class  파일이 실행되기 때문에 빠름
                            예시언어) C. C++, c#, Java
                            
        인터프리터 (interpreter): 한국어로 해석하다라는 의미
                                인터프리터는 소스 코드를 한 줄씩 해석하면서 바로 실행
                                코드를 작성한 다음 바로 실행 가능
                                한 줄씩 해석하면서 싱행하기 때문에 컴파일러 방식에 비해 속도가 느림
                                예시 언어)Python, JavaScript,    Ruby
